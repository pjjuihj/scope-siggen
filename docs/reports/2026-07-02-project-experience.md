## 做了什么

### 信号发生器

TIM5 定时器触发 DAC，DMA 自动把 256 点波形缓冲区搬到 DAC 寄存器，PA4 引脚输出模拟波形。支持 5 种波形（正弦/方波/三角/锯齿/直流），频率和幅度可调。

正弦波用 256 点查表实现，Flash 存储零 RAM 开销，比 `sinf()` 快 16 倍。

### 示波器

TIM8 定时器触发 ADC，DMA 循环采集 1024 点到双缓冲区。半传输/全传输中断通知 FreeRTOS 任务处理数据。

信号处理包括：过零检测算频率、平均值算电压、min/max 包络降采样显示。

触发系统用 ADC 模拟看门狗（AWD）实现硬件触发，1 ADC 时钟精度，CPU 开销极低。

### 波形显示

SSD1306 OLED 128x64，I2C 接口。波形渲染用 EMA 平滑缩放 + min/max 包络算法，无论波形有多少周期都能正确显示轮廓。

### 串口控制

21 条文本命令，中断接收环形缓冲区。命令分发用命令表（函数指针数组）替代 if-else 链，加新命令只需加一行表项。

### 配置管理

Flash Sector 11 存储配置，魔数 + 校验和双重验证。支持运行时修改、保存、掉电恢复。

## 难在哪

### 1. ADC 外部触发配置

STM32 的 ADC 外部触发模式有 4 个必要配置项，任何一个配错 ADC 就不转。默认值全部不对。

```c
ContinuousConvMode = DISABLE;     // 默认 ENABLE，必须改
ExternalTrigConv = TIM8_TRGO;     // 默认软件触发，必须改
DMAContinuousRequests = ENABLE;   // 默认 DISABLE，必须改
EOCSelection = SINGLE_CONV;       // 默认 SEQ_CONV，必须改
```

试错 2 小时没搞定，查 GitHub 5 分钟找到正确配置。

### 2. 启动顺序

DMA 和定时器的启动顺序决定成败。必须先启动 DMA，后启动定时器。反过来 DMA 不会开始传输。

### 3. 采样时间匹配

DAC 输出缓冲器阻抗约 15kΩ，ADC 采样时间必须用 480 cycles。用默认的 84 cycles，采样值只有 60% 正确，波形幅度严重偏低。

### 4. DMA 竞态条件

ISR 设置 `process_ptr` 通知任务有新数据，但没有内存屏障保护。任务可能读到旧指针或新旧混合数据。修复方法：volatile 声明 + 任务通知替代信号量。

### 5. 波形显示尖刺

128 像素宽度显示 1024 点数据，每像素 8 个采样点。如果取单点降采样，不同相位的点连起来就是尖刺。用 min/max 包络算法解决。

### 6. FreeRTOS 互斥锁

波形生成（768 次乘法）在锁内执行，阻塞其他任务读取配置。改为锁内拷贝配置、锁外生成波形、锁内应用硬件配置。

## 架构亮点

### 分层架构

应用层（示波器/信号发生器/显示/串口）→ 服务层（配置/错误追踪/环形缓冲区/调试）→ 驱动层（CubeMX 生成代码不修改）。

模块接口统一：`Init/Start/Stop/GetStatus/SelfTest`。

### 事件驱动任务模型

示波器任务用 `ulTaskNotifyTake` 等待 DMA 中断，不用轮询。显示任务用 dirty flag 减少无效刷新。串口用中断接收 + 环形缓冲区，不阻塞主循环。

### DMA 双缓冲

ADC DMA 循环写入 1024 点缓冲区，半传输/全传输回调通知任务处理已完成的一半。ISR 设置 `process_ptr`，任务读取后清 NULL。生产者-消费者模式，零拷贝。

## 学到了什么

### 寄存器不会骗人

软件层面猜来猜去，不如直接读硬件状态。读 `TIM8->CR2` 发现 TGRS=000，就知道 TRGO 没配置。读 `ADC->CR2` 发现 CONT=0，就知道连续模式没开。5 秒定位一个 bug。

### 先跑通再美化

第一天同时推进 4 个模块，结果 0 个功能验证通过。应该先用最丑的方式把一条链路跑通（DAC 输出 → 导线 → ADC 输入 → 串口打印），再加功能。

### 代码审查比写代码重要

每轮代码审查都发现了前一轮遗漏的问题。显示模块审查发现 8 项、触发系统 6 项、信号发生器 4 项。写代码是创造 bug 的过程，审查才是消灭 bug 的过程。

### AI 能写代码但不能验证

AI 可以生成看起来正确的初始化代码，但无法验证硬件是否真的工作。CubeMX 的默认值不适合外部触发模式，AI 不会知道，因为它无法烧录和测量。

- STM32 ADC 外部触发模式的 4 个必要配置

- OLED 波形渲染的 min/max 包络算法
