# 防骗机制文档

**日期**: 2026-06-29
**版本**: 1.0.0

---

## 1. 核心问题

AI 不会"遵守"文档，只会"生成"看起来符合文档的文本。

```
┌──────────────────────────────────────────────────────────────┐
│  AI 为什么会"骗人"？                                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. AI 不知道它不知道什么                                     │
│     └─ 它会自信地说"配置正确"，但没有验证                     │
│                                                              │
│  2. AI 没有硬件感知                                           │
│     └─ 它不知道寄存器实际值，只能猜                          │
│                                                              │
│  3. AI 倾向于"完成任务"                                      │
│     └─ 它会说"已完成"而不是"我做不到"                        │
│                                                              │
│  4. AI 没有验证机制                                           │
│     └─ 它说"正确"时，没有工具检查它是否真的正确              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 2. 五层防骗机制

```
┌──────────────────────────────────────────────────────────────┐
│  Layer 1: AI 不能说"我验证了"                                 │
│  ├─ AI 只能输出代码和预期值                                   │
│  ├─ AI 不能说"寄存器值是 xxx"（它不知道实际值）               │
│  └─ AI 必须说"预期值是 xxx，需要你验证"                       │
│                                                              │
│  Layer 2: AI 必须输出可执行的验证代码                         │
│  ├─ 不是"我验证了"，是"这是验证代码，你运行"                 │
│  ├─ 验证代码必须输出具体值（不是"正确"/"错误"）              │
│  └─ 验证代码必须可编译、可烧录、可运行                       │
│                                                              │
│  Layer 3: AI 必须输出检查清单                                 │
│  ├─ 不是"配置正确"，是"检查这 12 项"                         │
│  ├─ 检查清单必须具体（寄存器名+位域+预期值）                  │
│  └─ 检查清单必须可逐项打勾                                   │
│                                                              │
│  Layer 4: 用户必须确认                                        │
│  ├─ 关键步骤必须用户确认                                     │
│  ├─ 用户说"确认"才能继续                                     │
│  └─ 用户说"不对"必须回退                                     │
│                                                              │
│  Layer 5: 工具必须验证                                        │
│  ├─ 编译器验证（0 errors, 0 warnings）                       │
│  ├─ 静态分析验证（cppcheck）                                 │
│  ├─ 串口验证（读寄存器值）                                   │
│  └─ 万用表/示波器验证（硬件测量）                            │
│                                                              │
│  核心规则：AI 生成代码 + 预期值 + 验证代码                    │
│           人/工具执行验证                                     │
│           AI 不能代替验证                                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 3. 具体场景

### 3.1 AI 说"配置正确"

```
❌ AI 说："ADC 配置正确"

✅ AI 输出：
   代码：HAL_ADC_Start(&hadc1); ...
   预期值：ADC Raw 应在 506-2450 范围
   验证代码：
   void Verify_ADC(void) {
       HAL_ADC_Start(&hadc1);
       HAL_ADC_PollForConversion(&hadc1, 100);
       uint16_t val = HAL_ADC_GetValue(&hadc1);
       printf("ADC Raw=%d (%.3fV)\r\n", val, val*3.3f/4095);
   }
   用户：请运行 Verify_ADC() 并确认结果
```

### 3.2 AI 说"已修复"

```
❌ AI 说："已修复启动顺序问题"

✅ AI 输出：
   修改：oscilloscope.c 第 148-157 行
   修改前：HAL_TIM_Base_Start → HAL_ADC_Start_DMA
   修改后：HAL_ADC_Start_DMA → HAL_TIM_Base_Start
   验证：编译通过，检查启动顺序正确
   用户：请确认修改是否正确
```

### 3.3 AI 说"没有问题"

```
❌ AI 说："代码审查完成，没有问题"

✅ AI 输出：
   审查维度：C 语言安全
   检查项：6 项
   结果：6 项通过，0 项失败
   证据：每项检查的具体结果
   用户：请确认审查结果
```

## 4. 强制执行机制

### 4.1 状态机强制

```
statepilot 状态机：
├─ 不能跳过"读手册"直接写代码
├─ 不能跳过"验证"直接提交
├─ 不能跳过"代码审查"
└─ 失败必须回到修复（闭环）
```

### 4.2 Hook 拦截

```
Claude Code Hooks：
├─ PreToolUse: 检查是否允许操作
├─ PostToolUse: 验证结果是否符合预期
└─ 阻止非法操作
```

### 4.3 约束检查

```
约束检查：
├─ 写代码：最多 20 行，必须编译通过
├─ 修复：最多改 10 行，必须重新验证
└─ 验证：最多 3 次重试
```

## 5. 验证代码模板

### 5.1 ADC 验证

```c
void Verify_ADC(void) {
    printf("=== ADC Verification ===\r\n");

    // 启动 ADC
    HAL_ADC_Start(&hadc1);
    HAL_ADC_PollForConversion(&hadc1, 100);
    uint16_t val = HAL_ADC_GetValue(&hadc1);

    // 输出结果
    printf("ADC Raw=%d (%.3fV)\r\n", val, val*3.3f/4095);

    // 预期值
    printf("预期：值应随输入变化，不是全0或全4095\r\n");
}
```

### 5.2 TIM 验证

```c
void Verify_TIM(void) {
    printf("=== TIM Verification ===\r\n");

    // 读取计数器
    uint32_t cnt1 = TIM8->CNT;
    HAL_Delay(100);
    uint32_t cnt2 = TIM8->CNT;

    // 输出结果
    printf("TIM8 CNT: %lu -> %lu\r\n", cnt1, cnt2);

    // 预期值
    printf("预期：值应变化，不变化说明定时器没运行\r\n");
}
```

### 5.3 DMA 验证

```c
void Verify_DMA(void) {
    printf("=== DMA Verification ===\r\n");

    // 读取 DMA 状态
    uint32_t ndtr1 = DMA2_Stream0->NDTR;
    HAL_Delay(100);
    uint32_t ndtr2 = DMA2_Stream0->NDTR;

    // 输出结果
    printf("DMA NDTR: %lu -> %lu\r\n", ndtr1, ndtr2);

    // 预期值
    printf("预期：值应变化，不变化说明 DMA 没工作\r\n");
}
```

## 6. 检查清单模板

### 6.1 ADC 初始化检查清单

```
□ ADC 时钟使能（__HAL_RCC_ADC1_CLK_ENABLE）
□ GPIO 时钟使能（__HAL_RCC_GPIOA_CLK_ENABLE）
□ DMA 时钟使能（__HAL_RCC_DMA2_CLK_ENABLE）
□ GPIO 设为 ANALOG
□ DMA 流/通道配置正确
□ DMA 链接到 ADC（__HAL_LINKDMA）
□ DMA NVIC 使能
□ TIM TRGO = UPDATE
□ DMAContinuousRequests = ENABLE
□ EOCSelection = SINGLE_CONV
□ 采样时间足够（480 cycles）
□ 启动顺序正确（先 ADC DMA，后 TIM）
```
