### 1.2 外设约束

| 外设 | 引脚 | 配置 | 约束 |
|------|------|------|------|
| ADC1 | PA6 | TIM8 触发, DMA 循环 | 采样率 10kHz, 480 cycles |

## 2. 软件约束

### 2.1 库约束

| 库 | 版本 | 约束 |
|----|------|------|
| HAL | STM32F4xx | CubeMX 生成，不可手动修改 MSP |
| FreeRTOS | v10+ | CMSIS-RTOS v2 封装 |
| OLED | 自定义驱动 | I2C 接口，128x64 |

### 2.2 配置约束

| 配置项 | 值 | 说明 |
|--------|-----|------|
| ADC 采样时间 | 480 cycles | 高阻信号源必须 |
| DMAContinuousRequests | ENABLE | 循环采集必须 |
| EOCSelection | SINGLE_CONV | DMA 模式必须 |
| TIM TRGO | UPDATE | 不是 RESET |
| 启动顺序 | 先 ADC DMA，后 TIM | RM0090 要求 |

### 2.3 CubeMX 默认值陷阱

| 配置项 | 默认值 | 实际需要 | 后果 |
|--------|--------|---------|------|
| TIM_TRGO | RESET | UPDATE | 不输出触发信号 |
| DMAContinuousRequests | DISABLE | ENABLE | DMA 只跑一轮 |
| EOCSelection | SEQ_CONV | SINGLE_CONV | DMA 请求时机错误 |
| ADC 采样时间 | 84 cycles | 480 cycles | 高阻源读数偏小 |

## 3. 流程约束

### 3.1 开发流程约束

| 约束 | 说明 |
|------|------|
| 先读手册 | 写代码前必须读数据手册和库手册 |
| 每写 20 行就验证 | 不要写 400 行再测试 |
| 先硬件后软件 | 读寄存器判断硬件状态 |
| 库有 bug | 写代码前必须了解已知问题 |

### 3.2 验证流程约束

| 约束 | 说明 |
|------|------|
| Day 0: 读手册 | 必须先完成，不能跳过 |
| Day 1: 外设自检 | 每个外设单独验证 |
| Day 2: 数据链路 | 数据从 A 流到 B |
| Day 3: 集成显示 | 数据确认正确后才接 UI |

### 3.3 代码审查约束

| 维度 | 检查内容 |
|------|---------|
| C 语言安全 | 整数溢出、指针别名、未初始化变量 |
| 嵌入式专项 | 中断优先级、DMA 配置、时钟树 |
| 并发安全 | 竞态条件、内存屏障、volatile |

### 3.4 提交约束

| 约束 | 说明 |
|------|------|
| 提交格式 | type(scope): description |
| 硬件信息 | 必须包含 [hardware] 部分 |
| 测试结果 | 必须包含 [test] 部分 |
| 破坏性变更 | 必须包含 [breaking] 部分 |
