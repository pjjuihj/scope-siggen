# 上位机测试检查清单

## 测试前准备

### 硬件准备
- [ ] STM32F407 开发板
- [ ] USB 转串口模块
- [ ] 连接线（TX、RX、GND）
- [ ] 示波器探头（可选）
- [ ] 信号源（可选）

### 软件准备
- [ ] Keil MDK 已安装
- [ ] Node.js 已安装（v14+）
- [ ] Python 已安装（用于测试脚本）
- [ ] 浏览器（Chrome/Firefox/Edge）
- [ ] 串口调试助手（备用）

### 代码准备
- [ ] 已添加 `upper_computer.c` 到项目
- [ ] 已编译项目（0 errors, 0 warnings）
- [ ] 已烧录固件到开发板

## 测试步骤

### 1. 基本连接测试

#### 1.1 串口连接
- [ ] 连接 USB 转串口到开发板
  - TX → PA10 (USART1_RX)
  - RX → PA9 (USART1_TX)
  - GND → GND
- [ ] 检查串口驱动已安装
- [ ] 确认串口号（如 COM3）

#### 1.2 上位机启动
- [ ] 进入 `upper-computer` 目录
- [ ] 运行 `npm install`（首次）
- [ ] 运行 `npm start` 或双击 `start.bat`
- [ ] 打开浏览器访问 http://localhost:3000
- [ ] 确认页面正常显示

#### 1.3 连接测试
- [ ] 选择正确的串口
- [ ] 选择波特率 115200
- [ ] 点击"连接"按钮
- [ ] 确认状态显示"已连接"
- [ ] 确认日志显示连接成功

### 2. 命令测试

#### 2.1 基本命令
- [ ] 发送 `help` 命令
  - 预期: 显示帮助信息
- [ ] 发送 `status` 命令
  - 预期: 显示系统状态
- [ ] 发送 `version` 命令
  - 预期: 显示版本信息

#### 2.2 上位机命令
- [ ] 发送 `uc status` 命令
  - 预期: 显示上位机状态
- [ ] 发送 `uc test` 命令
  - 预期: 显示测试消息
- [ ] 发送 `uc send` 命令
  - 预期: 接收到波形数据

### 3. 数据传输测试

#### 3.1 手动发送
- [ ] 点击"开始采集"按钮
- [ ] 发送 `uc send` 命令
- [ ] 检查日志是否显示波形数据
- [ ] 确认数据格式正确（WAVE:...）

#### 3.2 自动流式传输
- [ ] 发送 `uc stream on` 命令
- [ ] 等待 2-3 秒
- [ ] 检查是否持续接收数据
- [ ] 确认数据更新频率正常
- [ ] 发送 `uc stream off` 命令
- [ ] 确认数据停止

#### 3.3 传输间隔测试
- [ ] 发送 `uc interval 50` 命令
- [ ] 启用流式传输
- [ ] 观察数据更新频率
- [ ] 确认间隔约为 50ms
- [ ] 恢复默认间隔

### 4. 波形显示测试

#### 4.1 波形绘制
- [ ] 启用流式传输
- [ ] 确认波形图表显示
- [ ] 确认波形更新流畅
- [ ] 确认自动缩放正常

#### 4.2 测量数据
- [ ] 确认频率显示正确
- [ ] 确认电压显示正确
- [ ] 确认峰峰值计算正确
- [ ] 确认最小/最大值正确

### 5. 信号发生器测试

#### 5.1 基本波形
- [ ] 选择正弦波
- [ ] 设置频率 1000 Hz
- [ ] 设置幅度 3.3 V
- [ ] 点击"启动信号源"
- [ ] 用示波器测量输出
- [ ] 确认波形正确

#### 5.2 不同波形
- [ ] 测试方波
- [ ] 测试三角波
- [ ] 测试锯齿波
- [ ] 测试直流

### 6. 界面功能测试

#### 6.1 控制按钮

## 常见问题

### 问题 1: 串口无法连接
**症状**: 点击连接后无反应
**解决**:
- [ ] 检查串口是否被其他程序占用
- [ ] 确认串口号正确
- [ ] 检查串口驱动

### 问题 2: 波形不显示
**症状**: 图表无数据
**解决**:
- [ ] 确认已点击"开始采集"
- [ ] 确认流式传输已启用
- [ ] 检查浏览器控制台错误

### 问题 3: 数据不同步
**症状**: 波形跳动或延迟
**解决**:
- [ ] 降低采样率
- [ ] 增加传输间隔
- [ ] 清除旧数据

### 问题 4: 命令无响应
**症状**: 发送命令后无反应
**解决**:
- [ ] 确认串口连接正常
- [ ] 检查命令格式
- [ ] 查看日志输出
