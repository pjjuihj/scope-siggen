# Domain Docs

## Layout

Single-context

## Files

- `CONTEXT.md` — repo root
- `docs/adr/` — Architecture Decision Records

## Consumer Rules

Skills (`improve-codebase-architecture`, `diagnosing-bugs`, `tdd`) read `CONTEXT.md` to learn the project's domain language. They read `docs/adr/` for past architectural decisions.

When contributing to this repo, update `CONTEXT.md` if domain terminology changes. Record significant architectural decisions in `docs/adr/`.
